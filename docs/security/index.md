# セキュリティ対策

システムを安全に運用するためのセキュリティ対策と運用プラクティスについて学びます。

## はじめに

セキュリティは、システム運用において最も重要な要素の一つです。しかし、**完全なセキュリティは存在しません**。重要なのは、リスクを適切に評価し、ビジネスの要件に応じて現実的な対策を講じることです。

このセクションでは、AWS環境を前提とした実践的なセキュリティ対策を学びます。技術的な対策だけでなく、コストとのバランス、ステークホルダーとのリスク合意など、実務で必要となる考え方も含まれています。

## なぜセキュリティを学ぶのか

現代のクラウド環境において、セキュリティは以下の理由から不可欠です:

- **ビジネスの継続性**: セキュリティインシデントは事業停止につながる
- **信頼の維持**: 顧客データの保護は企業の信頼の基盤
- **法的要件**: GDPR、個人情報保護法などへの対応義務
- **コスト削減**: 事前の対策は事後の対処より遥かに安価
- **競争優位性**: セキュアな環境は顧客選択の重要な要素

## セキュリティの現実

セキュリティ対策を進める上で、以下の現実を理解しておくことが重要です:

### 完全なセキュリティは存在しない

- どんな対策も突破される可能性がある
- 新しい脆弱性は日々発見される
- 攻撃者と防御者のいたちごっこは続く

これは業界標準のセキュリティフレームワークでも認識されている前提です：

- **NIST Cybersecurity Framework**: [Five Functions](https://www.nist.gov/cyberframework/online-learning/five-functions)が保護(Protect)だけでなく、検知(Detect)・対応(Respond)・復旧(Recover)を含むことは、侵害の発生を前提としたリスクベースのアプローチを示している

### リスク許容の判断が必要

- すべてのリスクを排除することは不可能
- ビジネス要件と照らし合わせた優先順位付けが必要
- どこまでリスクを許容するかの判断が求められる

### ステークホルダーとの合意が不可欠

- 技術者だけでリスク判断はできない
- ビジネス側との共通理解が必要
- 受容するリスクは文書化し、定期的に見直す
