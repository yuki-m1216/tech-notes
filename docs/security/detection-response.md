# 検知と対応

**📝 このページは現在編集中です**

このセクションでは、セキュリティイベントの検知からインシデント対応までの一連の流れについて学びます。

## 予定している内容

### なぜ検知が重要か

- 予防だけでは不十分な理由
- 検知から対応までの流れ
- 早期検知がもたらすビジネス価値
- NIST CSF の Detect → Respond → Recover

### ログ監視の考え方

- 何をログに記録すべきか
- 記録すべき情報と記録してはいけない情報
- ログの保持期間の考え方
- ログの改ざん防止

### 異常検知とアラート

- ベースライン監視とアノマリ検知
- 相関分析の重要性
- アラート疲れを防ぐ設計
- 効果的なアラートの条件

### インシデント対応の重要性

- なぜ事前準備が必要か
- インシデント対応とビジネス継続性
- 初動対応の遅れがもたらす影響

### インシデント対応の流れ

- 準備 (Preparation)
- 検知と分析 (Detection & Analysis)
- 封じ込め・根絶・復旧 (Containment, Eradication & Recovery)
- 事後対応 (Post-Incident Activity)

### インシデント対応計画

- 何を事前に決めておくべきか
- 役割と責任の明確化
- エスカレーションパスの設計
- コミュニケーション計画

### AWS環境での実装

- CloudTrail・GuardDuty・Security Hubの役割分担
- ログの集約と長期保存
- セキュリティアラートの設計原則
- インシデント対応を支援するAWSサービス
- よくある失敗パターン
